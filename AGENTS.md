## 3. Sync model — git through GitHub (NOT file-copy)

The canonical source of truth is the **private GitHub repo** `sagedeutschle/kaleidoscope`.
Three clones stay in sync through it:

```
   Sage's desktop  ┐
   Ben's machine   ┼──►  GitHub (canonical)  ◄── NAS shared folder ("the bin")
   NAS clone       ┘         push / pull
```

**Every session:**
1. `git pull --rebase` **before** you start. Read the latest `docs/AGENT-COORDINATION.md` entries.
2. Work in **small, frequent commits** with clear messages.
3. `git push` when you finish a coherent unit. Don't sit on large uncommitted trees.
4. If a push is rejected, `git pull --rebase` and resolve, then push again.

Helper scripts: `scripts/pull.sh`, `scripts/push.sh`, `scripts/sync.sh` (pull-rebase-then-push).

**Do not** mirror by rsync/Finder-copy anymore. The old `ios/scripts/sync-shared-folder.sh`
(rsync to the NAS) is **deprecated** — git replaces it.

## 4. Build rules (hard-won — violating these wastes hours)

1. **Build on your LOCAL clone, never on the NAS/SMB mount.** SMB + Xcode = signing and
   xattr failures. The NAS clone is for browsing/backup, not building.
   Also: the SMB mount mangles file modes, so git on the NAS clone reports phantom
   "unstaged changes" — its config carries `core.fileMode false` for this; set it again
   if a fresh NAS clone is ever made. Git ops there are SLOW (minutes); be patient or
   run them in the background.
2. **Derived data goes under `~/Library/Caches/…`, never inside the project.** Building into
   an in-project path triggers iCloud/file-provider xattrs → `CodeSign failed`.
   Example: `-derivedDataPath ~/Library/Caches/KaleidoscopeBuild`.
3. **`.xcodeproj` is generated from `project.yml`** and is **gitignored**. After you clone
   or pull, run `xcodegen generate`. **Never hand-edit the `.xcodeproj`.**
4. **`Info.plist` is regenerated by xcodegen** from `project.yml` (`targets.<T>.info.properties`).
   Put plist keys there. Hand-edits to `Info.plist` and `INFOPLIST_KEY_*` build settings are
   silently lost.
5. **Release archives use `SWIFT_COMPILATION_MODE=incremental`** — whole-module optimization
   crashes swift-frontend on these targets.
6. **iOS device build+deploy is delegated to the `codex` CLI** (established workhorse;
   binary `/Applications/Codex.app/Contents/Resources/codex`). Fallback when codex is
   unavailable: `xcodebuild` then `xcrun devicectl device install/launch --device <CoreDevice-id>`.
7. **Two device identifiers differ:** `xcodebuild -destination 'id=…'` wants the **hardware UDID**;
   `devicectl` wants the **CoreDevice id**. See §6.

## 5. Coordination protocol (PRISM) — how we don't clobber each other

`docs/AGENT-COORDINATION.md` is the shared ledger. Before touching code:

1. **Claim** — add a dated entry naming your lane + the files you'll touch.
2. **Sweep before building** — another agent may have edited a shared file since your last
   pull. `git pull --rebase`, then re-read files you're about to build; grep for others'
   claim markers. Files move — use `find`, not assumed paths.
3. **Log** — after finishing, record exactly what you changed and the build/test status.
4. **Lanes:** Codex = wiring/function/backend/data; Claude & Fable = design/polish/visual.
   You may work vertically, but announce it. When two agents share a repo, whoever touches
   `HomeView` / catalogs / `project.yml` says so loudly (those are collision hotspots).
5. Card `id` strings and asset ids are **contracts** — never rename them; routes and saved
   data key on them.

**Common commands** (run from a local clone)
```bash
# iOS
cd ios && xcodegen generate
# sim build
xcodebuild -project Kaleidoscope.xcodeproj -scheme Kaleidoscope \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -derivedDataPath ~/Library/Caches/KaleidoscopeBuild build
# device deploy → prefer codex (see §4.6); tests:
xcodebuild test -project Kaleidoscope.xcodeproj -scheme Kaleidoscope \
  -destination 'platform=iOS Simulator,name=iPhone 17' -derivedDataPath ~/Library/Caches/KaleidoscopeBuild

# macOS
cd macos && xcodegen generate && ./scripts/deploy-mac.sh   # installs to ~/Applications

# Oracle (single-writer = Sage's laptop only)
cd oracle && python3 -m wkd --config config.fun.json run && python3 tools/export_decrees_json.py fun.db decrees.json
```

  `ios/docs/Secrets.example.swift` and fill in. New collaborators get real values from Sage **out of band**.
